👋 认识 GDScript
你已经懂变量和函数了，那你已经有了最重要的基础！今天只需要学 GDScript 和你以前学过的有什么不一样。
GDScript 是 Godot 游戏引擎专用的编程语言，长得很像 Python。你学过的变量、函数、if、for 这里全都能用，只是语法略有不同。
普通编程

代码跑完就结束
你自己画界面
函数要自己调用

GDScript

游戏一直运行，等玩家操作
Godot 帮你管理按钮文字
有些函数 Godot 自动调用

💭 思考一下（不计分）
你学过的编程语言里，写好程序运行一次就结束了。但游戏不一样——你打开游戏，它会一直等你操作。你觉得游戏为什么能一直等着，不会自己结束？
游戏有一个不停运行的「主循环」，就像一个永不停止的 for 循环，每一帧都在检查：玩家有没有按键？有没有点击？根据玩家的操作再做出反应。Godot 帮我们管理了这个主循环，我们只需要写「玩家操作时应该发生什么」。

📄 猜数字游戏的变量
点击每行看说明
1
var secret_number = 0
建一个叫 secret_number 的盒子，初始放 0。
游戏开始时会换成随机数。var 是 GDScript 建变量必须写的关键字。
2
var remaining_lives = 5
剩余猜测次数，初始 5。每猜错一次减 1，减到 0 游戏结束。
remaining = 剩余，lives = 生命/次数。
3
var score = 100
玩家分数，初始 100 分。每猜错扣 20 分，猜对了保留当前分数。越早猜对，分越高！
4
var game_over = false

注意 GDScript 用小写 true/false，不像有些语言用大写 True/False！
💭 先想一想
为什么需要 game_over 这个变量？如果没有它，游戏结束后会发生什么问题？
如果没有 game_over，游戏结束后玩家还能继续猜！比如已经猜对了，但玩家还在输入数字，游戏就乱了。
game_over 就是一道「门」，游戏结束就把门关上，不让玩家继续操作。

第 3 / 8 步
⚙️ 函数 — GDScript 写法
函数就是「把一组步骤打包起来，起个名字」。GDScript 用 func 关键字定义函数。
GDScript 最容易忘的两件事：
1. 函数名后面的冒号 : 不能漏掉
2. 函数体必须缩进（用 Tab 键）
📄 游戏的两个核心函数
点击每行看说明
1
func _ready():
_ready 是 Godot 的特殊函数，场景加载完自动运行一次。名字前的 _ 代表「系统函数」，不需要你手动调用。
我们在这里生成随机数，初始化游戏。
2
secret_number = randi_range(1, 100)
randi_range(1, 100) 是 GDScript 内建函数，生成 1 到 100 的随机整数。就像让电脑闭着眼睛随机想一个数字。
缩进用 Tab 键，代表这行属于 _ready 函数。
4
func check_guess(player_input):
check_guess 是我们自己定义的函数，玩家点击「确认」时调用它。
player_input 是参数，玩家输入的数字通过这里传进来。就像点外卖时填「备注」，函数拿到备注再做对应处理。
5
if game_over: return
return 是「立刻停止，退出这个函数」。游戏已经结束了，就不继续处理玩家的猜测。就像门已经关了，新来的人不能进去。
💭 先想一想
游戏里有 _ready() 和 check_guess() 两个函数。谁来调用 _ready()？谁来调用 check_guess()？时机有什么不同？
_ready()：Godot 系统自动调用，游戏场景加载好时运行，只跑一次。你不需要写任何代码来调用它。

check_guess()：玩家点击「确认」按钮时，按钮的 pressed 信号触发，调用这个函数。每次玩家猜测都会调用一次，可以调用很多次。

第 4 / 8 步
🔀 条件判断 if / elif / else
你应该已经认识 if/else 了！GDScript 写法几乎一样，只记住：条件后面要加冒号 :，下一行要缩进。
📄 猜数字的核心判断
点击每行看说明
1
if player_input == secret_number:
== 是「等于」的比较（不是赋值的单个 =）。检查玩家输入的数是不是和秘密数字相同。
冒号 : 是必须的，代表「下面缩进的是这个条件成立时要做的事」。
2
game_over = true
猜对了！把游戏结束的开关打开。
3
elif player_input < secret_number:
elif = else if（否则如果）。上面条件不满足时才判断这个。
< 是小于号。玩家猜的数比秘密数小 = 猜小了。
4
remaining_lives -= 1
-= 1 是「在原来基础上减 1」的简写，等同于 remaining_lives = remaining_lives - 1。猜错一次，次数少一次。
5
else:
以上条件都不满足 = 玩家猜的数比秘密数大 = 猜大了。else 不需要写条件，直接处理「剩余所有情况」。
6
if remaining_lives == 0 and not game_over:
检查次数用完且没赢。
and = 两个条件都要满足
not = 「不是」的意思
所以这行 = 「次数是0，而且还没赢」才进入这个分支。

第 5 / 8 步
🖥️ 输出到游戏界面
普通编程用 print() 显示文字。但 print() 只在调试面板出现，玩家看不到。Godot 里要让玩家看到文字，要改节点的 .text 属性。
调试用（玩家看不到）

print("猜对了！")

只在 Godot Output 面板显示

界面显示（玩家能看到）

$StatusLabel.text = "猜对了！"

$ 找到节点，.text 改文字

📄 字符串拼接：数字 + 文字
点击看说明
1
var score = 80
先建一个分数变量。
2
$ScoreLabel.text = "你的分数：" + str(score)
把文字和数字拼在一起显示「你的分数：80」。
str(score) 把数字变成文字，因为文字和数字不能直接用 + 拼接。就像字母和数字不能直接加，要先「翻译」成同一种格式。
3
$ScoreLabel.text = "你的分数：%d" % score
另一种更简洁的写法：用 %d 占位（d 代表整数），% score 把数字填进去。
结果一样是「你的分数：80」，但写起来更整洁。
💭 先想一想
如果想在游戏界面显示「游戏结束！答案是 63」，其中 63 是存在变量 secret_number 里的，你会怎么写这行代码？
有两种写法都对：

$ResultLabel.text = "游戏结束！答案是 " + str(secret_number)

或者更简洁的：

$ResultLabel.text = "游戏结束！答案是 %d" % secret_number
